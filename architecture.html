Data Flow

User Interaction → Rendered UI

1. UserChanges a filter or navigates to a view
→
2. App.vueUpdates reactive filter refs (selectedPeriod, selectedWarehouse…)
→
3. watch()View watches filter props, calls fetchData()
→
4. api.jsBuilds URLSearchParams, calls axios.get()
→
5. FastAPIReads query params, runs filter helpers
→
6. mock_dataReturns filtered Python list slice
→
7. PydanticValidates & serialises to JSON
→
8. VueStores in ref, computed properties derive summaries, v-for renders

Reactivity pattern
Raw fetched data lives in ref([]) (allOrders, inventoryItems, …).
Derived values (summaries, sorted tables, search results) are computed() — they re-run automatically when the raw ref changes.
Re-fetches only happen when a filter changes, not on every computed access.

Filter System

Four global filters live in App.vue and are passed as props to every view. The backend skips any filter set to 'all'.

Time Period
param: month
Values: all, 2025-01 … 2025-12, Q1-2025 … Q4-2025
Match: substring in date field
Warehouse
param: warehouse
Values: all, North, South, East, West
Match: exact (case-sensitive)
Category
param: category
Values: all, Electronics, Mechanical, Raw Materials, …
Match: case-insensitive
Order Status
param: status
Values: all, Pending, Fulfilled, Cancelled
Match: case-insensitive
